# 🌥️ Clima

## ☀️ async
- 비동기 함수 정의시 사용
- 함수 안에서 시간이 걸리는 작업(API 호출, 위치 정보 가져오기 등)을 기다리지 않고 다른 코드가 먼저 실행될 수 있게 함
- await 와 함께 쓰임

## 🌧️ await
- async 함수 안에서 사용되어 특정 작업이 끝날 때까지 기다리게 함
- await Geolocator.getCurrentPosition() 은 위치는 받아질 때까지 기다린 후 다음 코드로 넘어감
- CPU가 멈추는 게 아니라 그동안 다른 작업을 계속할 수 있는 비동기 처리

## ☂️ Future
- 미래에 완료될 작업의 결과를 표현하는 객체
- Future<void> 는 나중에 완료되지만 값을 반환하지 않는 작업을 의미
- await 으로 Future의 결과를 받아오거나, .then() 으로 완료 후 실행할 코드 지정 가능

## 🌈 initState
- StatefulWidget 이 생성될 때 처음 한 번만 호출됨
- 화면이 그려지기 전에 데이터를 불러오거나 초기 설정할 때 사용
- initState() 에서 getLocation( ) 을 호출해 위치 정보를 미리 불러옴

## 🌨️ build
- 위젯이 화면에 표시될 때마다 호출되는 함수
- UI를 그리는 역할을 하며 항상 Widget을 반환
- 상태가 바뀌면(setState) 자동으로 다시 실행돼서 화면이 갱신됨

## ☃️ deactivate
- 위젯이 화면에서 제거되기 직전에 호출됨
- 보통 리소스 해제, 혹은 상태 저장시 사용
- 예를 들어 화면 전환 시 기존 화면이 사라질 때 실행됨

## ❄️ try and catch
- 에러가 발생할 수 있는 코드를 try 안에 넣고, 문제가 생기면 catch(e) 가 실행돼 에러를 안전하게 처리
- 예: 위치를 불러올 때 GPS 오류가 나면 앱이 멈추지 않고 print(e) 로 에러만 출력됨

## 🌙 null aware operator (?., ??, ??=)
- 값이 null인지 확인할 때 사용하는 연산자
- a?.b -> a 가 null 이면 오류 없이 null 반환
- x ?? y -> x 가 null 이면 y 를 대신 사용
- z ??= 5 -> z 가 null 일 때만 5로 초기화

## 🔥 API
- API(Application Programming Interface)는 앱과 서버가 데이터를 주고받는 방법
- OpenWeatherMap API를 통해 온도나 날씨 데이터를 받아올 수 있음
- http.get() 요청으로 데이터를 가져와 JSON 형식으로 해석해 사용

## ⛱️ status code
- 서버 응답 결과를 나타내는 숫자 코드
- 200은 성공, 404는 페이지 없음, 500은 서버 오류를 뜻함
- if(response.statusCode == 200) 처럼 사용해 요청 성공 여부 확인 가능
